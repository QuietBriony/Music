# Xtal Reference Gradient Depth Map

## Purpose

This document deepens the 16-track Xtal-centered reference list into listening axes for future Music tuning.

It does not add audio, samples, lyrics, Apple Music URLs, preview URLs, or copied musical material. The goal is to improve how references are translated into Music-specific production parameters.

## Depth Axes

Use these axes while listening and reviewing m4a recordings:

- foreground vs bed: what should speak clearly, and what should only stain the room
- pulse visibility: whether motion is beat, breath, dust, or delayed memory
- particle material: glass, dust, pluck, tape, chrome, ghost, or pressure
- low-mid behavior: warm body, murk, hollow room, or cleaned-out air
- tail behavior: short smear, long bloom, cold distance, or wet interior
- gesture role: what faders and momentary pads should do with the reference

## Engine Translation Priorities

1. Keep a continuous bed so the rig does not feel like isolated events.
2. Make pads audible through texture identity, not only volume.
3. Use transparent particles above 2kHz carefully; avoid hard click.
4. Keep low-mid body alive but cleaned out.
5. Let AutoMix move the gradient slowly while faders bend it by hand.
6. Make iOS Safari playback reliability part of every recording review.

## Suggested Next Listening Pass

Use iPhone or iPad Safari if possible.

1. `START`, `AUTO MIX` on, `OUTPUT` 75 to 85.
2. Listen for 30 seconds without touching.
3. Switch apps for at least 60 seconds.
4. Return to Safari.
5. Hold DRIFT, REPEAT, PUNCH, VOID in sequence.
6. Record a 30 second m4a if playback stayed stable.
7. Review whether the bed, particles, and pad identities match the depth axes above.
